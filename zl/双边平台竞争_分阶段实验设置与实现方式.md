## 1. 统一基础模型

后续实验都可以基于同一个动态模型展开。

### 1.1 状态变量

设两个平台为平台 A 和平台 B。用户侧和商户侧均采用单归属假设，即每个用户或商户主要选择一个平台。

令：

$$
u(t)=u_A(t)$$

表示平台 A 的用户份额，则平台 B 的用户份额为：

$$1-u(t)$$

令：

$$m(t)=m_A(t)$$

表示平台 A 的商户份额，则平台 B 的商户份额为：

$$1-m(t)$$

因此，模型只需要跟踪两个状态变量：

$$u(t),\quad m(t)$$

### 1.2 效用函数

用户选择平台 A 的效用：

$$
V_A^U=q_A^U+\alpha m+b_A^U-p_A^U+s_Uu
$$

用户选择平台 B 的效用：

$$
V_B^U=q_B^U+\alpha(1-m)+b_B^U-p_B^U+s_U(1-u)
$$

商户选择平台 A 的效用：

$$
V_A^M=q_A^M+\beta u+b_A^M-r_A+s_Mm
$$

商户选择平台 B 的效用：

$$
V_B^M=q_B^M+\beta(1-u)+b_B^M-r_B+s_M(1-m)
$$

符号含义如下：

| 符号 | 含义 |
|---|---|
| $u(t)$ | 平台 A 的用户份额 |
| $m(t)$ | 平台 A 的商户份额 |
| $q_A^U,q_B^U$ | 平台 A/B 对用户侧的服务质量 |
| $q_A^M,q_B^M$ | 平台 A/B 对商户侧的服务质量 |
| $\alpha$ | 商户规模对用户吸引力的影响强度 |
| $\beta$ | 用户规模对商户吸引力的影响强度 |
| $b_A^U,b_A^M$ | 平台 A 对用户/商户的补贴 |
| $p_A^U,p_B^U$ | 用户侧使用成本或价格 |
| $r_A,r_B$ | 平台对商户的抽成或使用成本 |
| $s_U,s_M$ | 用户/商户侧切换成本或锁定效应 |

### 1.3 Logit 选择概率

用户选择平台 A 的概率：

$$
P_A^U=\frac{e^{\gamma_U V_A^U}}{e^{\gamma_U V_A^U}+e^{\gamma_U V_B^U}}
$$

商户选择平台 A 的概率：

$$
P_A^M=\frac{e^{\gamma_M V_A^M}}{e^{\gamma_M V_A^M}+e^{\gamma_M V_B^M}}
$$

其中，$\gamma_U,\gamma_M$ 表示用户和商户对效用差异的敏感程度。

### 1.4 动态演化方程

$$
\frac{du}{dt}=\lambda_U(P_A^U-u)
$$

$$
\frac{dm}{dt}=\lambda_M(P_A^M-m)
$$

含义是：如果根据当前效用计算出的平台 A 吸引概率高于当前份额，则平台 A 份额上升；反之下降。

### 1.5 基础实现方式

Python 中可以使用：

```python
import numpy as np
from scipy.integrate import solve_ivp


def platform_dynamics(t, y, params):
    u, m = y
    alpha = params["alpha"]
    beta = params["beta"]
    sU = params["sU"]
    sM = params["sM"]
    gammaU = params["gammaU"]
    gammaM = params["gammaM"]
    lambdaU = params["lambdaU"]
    lambdaM = params["lambdaM"]

    qAU = params["qAU"]
    qBU = params["qBU"]
    qAM = params["qAM"]
    qBM = params["qBM"]
    bAU = params["bAU"]
    bBU = params["bBU"]
    bAM = params["bAM"]
    bBM = params["bBM"]
    pAU = params["pAU"]
    pBU = params["pBU"]
    rA = params["rA"]
    rB = params["rB"]

    VAU = qAU + alpha * m + bAU - pAU + sU * u
    VBU = qBU + alpha * (1 - m) + bBU - pBU + sU * (1 - u)
    VAM = qAM + beta * u + bAM - rA + sM * m
    VBM = qBM + beta * (1 - u) + bBM - rB + sM * (1 - m)

    expAU = np.exp(gammaU * VAU)
    expBU = np.exp(gammaU * VBU)
    expAM = np.exp(gammaM * VAM)
    expBM = np.exp(gammaM * VBM)

    PAU = expAU / (expAU + expBU)
    PAM = expAM / (expAM + expBM)

    du = lambdaU * (PAU - u)
    dm = lambdaM * (PAM - m)
    return [du, dm]


def run_simulation(params, u0=0.5, m0=0.5, T=50):
    sol = solve_ivp(
        fun=lambda t, y: platform_dynamics(t, y, params),
        t_span=(0, T),
        y0=[u0, m0],
        t_eval=np.linspace(0, T, 500),
        rtol=1e-8,
        atol=1e-8
    )
    return sol.t, sol.y[0], sol.y[1]
```

# 第一阶段：基础必做实验

## 实验 1：基础动态演化实验

### 研究问题

两个竞争平台在双边网络效应下，用户份额和商户份额如何随时间变化？系统最终会走向平台共存、市场倾斜还是赢家通吃？

### 实验设置

设置一组基础参数：

| 参数 | 建议值 | 含义 |
|---|---:|---|
| $\alpha$ | 1.0 | 商户对用户的吸引强度 |
| $\beta$ | 1.0 | 用户对商户的吸引强度 |
| $q_A^U=q_B^U$ | 0 | 两平台用户侧服务质量相同 |
| $q_A^M=q_B^M$ | 0 | 两平台商户侧服务质量相同 |
| $b_A^U=b_A^M=b_B^U=b_B^M$ | 0 | 暂不考虑补贴 |
| $s_U=s_M$ | 0.2 | 较弱锁定效应 |
| $\gamma_U=\gamma_M$ | 2.0 | 选择敏感程度 |
| $\lambda_U=\lambda_M$ | 1.0 | 调整速度 |

初始条件设置三组：

| 组别 | $u_A(0)$ | $m_A(0)$ | 目的 |
|---|---:|---:|---|
| 对称初始 | 0.5 | 0.5 | 验证完全对称时是否共存 |
| 轻微领先 | 0.55 | 0.55 | 观察小优势是否被放大 |
| 明显领先 | 0.7 | 0.7 | 观察强优势是否导致锁定 |

### 实现方式

1. 固定基础参数；
2. 分别运行三组初始条件；
3. 记录 $u(t)$ 和 $m(t)$；
4. 绘制时间演化曲线。

### 展示结果

建议展示两张图：

1. 平台 A 用户份额 $u(t)$ 随时间变化；
2. 平台 A 商户份额 $m(t)$ 随时间变化。

可以在同一张图中展示不同初始条件。

### 结果解释重点

需要说明：

- 对称初始条件下，若两个平台完全相同，系统可能维持均衡；
- 平台 A 初始领先时，领先是否被放大取决于网络效应强度；
- 如果最终 $u,m$ 接近 1，说明平台 A 形成市场锁定；如果接近 0.5，说明平台共存。

## 实验 2：双边网络效应强度实验

### 研究问题

双边网络效应是否会导致市场锁定？网络效应强度达到什么程度时，市场从共存转向赢家通吃？

### 实验设置

扫描参数：

$$
\alpha \in [0,5],\quad \beta \in [0,5]
$$

例如各取 51 个点，形成 $51\times 51$ 的参数网格。

固定其他参数：

| 参数 | 建议值 |
|---|---:|
| $u_A(0)$ | 0.55 |
| $m_A(0)$ | 0.55 |
| $s_U=s_M$ | 0.2 |
| $q_A=q_B$ | 0 |
| 补贴 | 0 |

### 指标定义

市场集中度指标：

$$
C=|u_A(\infty)-0.5|+|m_A(\infty)-0.5|
$$

判断标准可以设为：

| 市场状态 | 判断条件 |
|---|---|
| 双平台共存 | $C<0.2$ |
| 市场倾斜 | $0.2\le C<0.8$ |
| 市场锁定 | $C\ge 0.8$ |

也可以定义平台 A 锁定：

$$
u_A(\infty)>0.8,\quad m_A(\infty)>0.8$$

### 实现方式

伪代码：

```python
alphas = np.linspace(0, 5, 51)
betas = np.linspace(0, 5, 51)
C_grid = np.zeros((len(betas), len(alphas)))

for i, beta in enumerate(betas):
    for j, alpha in enumerate(alphas):
        params["alpha"] = alpha
        params["beta"] = beta
        t, u, m = run_simulation(params, u0=0.55, m0=0.55)
        u_inf = u[-1]
        m_inf = m[-1]
        C_grid[i, j] = abs(u_inf - 0.5) + abs(m_inf - 0.5)
```

### 展示结果

绘制热力图：

- 横轴：$\alpha$；
- 纵轴：$\beta$；
- 颜色：市场集中度 $C$。

### 结果解释重点

需要重点说明：

- 低 $\alpha,\beta$ 区域对应双平台共存；
- 高 $\alpha,\beta$ 区域对应赢家通吃；
- 当只有一侧网络效应强时，市场可能出现不完全锁定；
- 双边网络效应越强，小初始优势越容易被放大。

## 实验 3：初始规模差异实验

### 研究问题

平台初始规模差异是否会被网络效应放大？早期优势是否会导致长期市场锁定？

### 实验设置

固定网络效应为两种水平：

| 组别 | $\alpha$ | $\beta$ | 含义 |
|---|---:|---:|---|
| 弱网络效应 | 0.5 | 0.5 | 正反馈弱 |
| 强网络效应 | 3.0 | 3.0 | 正反馈强 |

扫描初始份额：

$$
u_A(0)=m_A(0)=x,\quad x\in[0.1,0.9]
$$

### 实现方式

1. 对每个初始份额 $x$，运行一次仿真；
2. 记录最终 $u_A(\infty)$ 和 $m_A(\infty)$；
3. 分别在弱网络效应和强网络效应下重复实验。

### 展示结果

建议画：

- 横轴：初始份额 $x$；
- 纵轴：最终用户份额 $u_A(\infty)$ 或综合份额 $L_A$。

其中综合份额定义为：

$$
L_A=\frac{u_A(\infty)+m_A(\infty)}{2}
$$

### 结果解释重点

重点说明：

- 弱网络效应下，初始差异可能不会被明显放大；
- 强网络效应下，初始份额略高于 0.5 就可能走向锁定；
- 这说明双边平台竞争存在路径依赖，早期市场进入和规模积累非常重要。

## 实验 4：服务质量差异实验

### 研究问题

如果平台 A 初始落后，但服务质量更高，它能否逆袭平台 B？逆袭需要多大的服务质量优势？

### 实验设置

设平台 A 初始落后：

$$
u_A(0)=0.3,\quad m_A(0)=0.3$$

平台 B 初始领先：

$$
u_B(0)=0.7,\quad m_B(0)=0.7$$

设置平台 A 的服务质量优势：

$$
\Delta q=q_A-q_B
$$

扫描：

$$
\Delta q\in[0,5]
$$

可以令：

$$
q_A^U=q_A^M=\Delta q,\quad q_B^U=q_B^M=0
$$

### 逆袭判断标准

平台 A 逆袭成功条件：

$$
u_A(\infty)>0.5,\quad m_A(\infty)>0.5$$

或者更严格地定义为：

$$
L_A>0.6
$$

### 实现方式

1. 固定平台 A 初始落后；
2. 扫描 $\Delta q$；
3. 对每个 $\Delta q$ 运行仿真；
4. 找到最小逆袭阈值 $\Delta q^*$。

### 展示结果

建议画：

- 横轴：服务质量优势 $\Delta q$；
- 纵轴：平台 A 最终综合份额 $L_A$；
- 用竖线标出逆袭阈值 $\Delta q^*$。

### 结果解释重点

可以写出：

- 服务质量优势确实能提高平台 A 的吸引力；
- 但在强网络效应和高切换成本下，弱势平台需要超过一定质量阈值才能逆袭；
- 因此，后发平台不能只依靠“小幅质量提升”，需要形成显著差异化优势。

## 实验 5：补贴策略基础实验

### 研究问题

补贴能否帮助弱势平台打破市场锁定？补贴是否越高越好？

### 实验设置

设平台 A 初始落后：

$$
u_A(0)=0.3,\quad m_A(0)=0.3$$

平台 A 提供统一补贴：

$$
b_A^U=b_A^M=b
$$

扫描：

$$
b\in[0,5]
$$

平台 B 不补贴：

$$
b_B^U=b_B^M=0
$$

### 实现方式

1. 固定其他参数；
2. 扫描补贴强度 $b$；
3. 记录最终市场份额 $L_A$；
4. 观察补贴与最终份额之间的关系。

### 展示结果

画两类图：

1. 不同补贴强度下的 $u(t),m(t)$ 时间曲线；
2. 横轴为补贴强度 $b$，纵轴为最终综合份额 $L_A$ 的曲线。

### 结果解释重点

需要避免简单结论“补贴越高越好”，而应说明：

- 低补贴可能只能带来短期增长，无法突破锁定；
- 中等补贴可能帮助平台跨过临界规模；
- 高补贴虽然提高份额，但若考虑成本，未必是最优策略。

## 实验 7：弱势平台逆袭阈值的影响因素实验

$$
u_A(0)=m_A(0)=0.3$$

对于不同网络效应强度：

\alpha=\beta\in\{0.5,1.0,2.0,3.0,4.0\}

对于不同切换成本：

s_U=s_M\in\{0,0.5,1.0,1.5\}

分别扫描服务质量优势：

$$
\Delta q\in[0,5]

找到最小逆袭阈值 $\Delta q^*$。

伪代码：

```python
thresholds = []
for k in network_strengths:
    for s in switching_costs:
```

## 实验 8：双边补贴分配实验

### 研究问题

在总预算固定时，平台应该补贴用户，补贴商户，还是双边补贴？

### 实验设置

设平台 A 初始落后：

$$
u_A(0)=m_A(0)=0.3$$

总补贴预算：

$$
B=b_A^U+b_A^M
$$

设：

$$
b_A^U=\rho B
$$

$$
b_A^M=(1-\rho)B
$$

扫描：

$$
\rho\in[0,1]
$$

其中：

- $\rho=1$：只补贴用户；
- $\rho=0$：只补贴商户；
- $0<\rho<1$：双边补贴。

### 实现方式

1. 固定总预算 $B$；
2. 扫描补贴分配比例 $\rho$；
3. 运行仿真；
4. 记录最终综合份额 $L_A$；
5. 找到最优补贴比例 $\rho^*$。

### 展示结果

建议画：

- 横轴：用户补贴比例 $\rho$；
- 纵轴：最终综合份额 $L_A$；
- 标出最优 $\rho^*$。

可以进一步画不同 $B$ 下的多条曲线。

### 结果解释重点

可以根据 $\alpha,\beta$ 的相对大小解释：

- 当用户更看重商户丰富度，即 $\alpha$ 较大时，补贴商户可能更有效；
- 当商户更看重用户规模，即 $\beta$ 较大时，补贴用户可能更有效；
- 当两侧网络效应都强时，双边补贴可能更容易跨过临界规模。

## 实验 9：阶段性补贴退出实验

### 研究问题

平台是否需要长期补贴？补贴结束后，平台能否依靠已经形成的网络效应维持优势？

### 实验设置

设补贴为时间函数：

$$
b(t)=
\begin{cases}
 b_0, & 0\le t\le T_s \\
 0, & t>T_s
\end{cases}
$$

其中：

- $b_0$ 是补贴强度；
- $T_s$ 是补贴持续时间。

扫描：

$$
b_0\in[0,5]
$$

$$
T_s\in[0,30]
$$

### 实现方式

动态函数中需要让补贴依赖时间：

```python
def get_subsidy(t, b0, Ts):
    return b0 if t <= Ts else 0.0
```

在 `platform_dynamics` 中，根据当前时间 $t$ 动态设置：

```python
bAU = get_subsidy(t, b0, Ts)
bAM = get_subsidy(t, b0, Ts)
```

然后对 $b_0,T_s$ 进行二维扫描。

### 展示结果

画热力图：

- 横轴：补贴强度 $b_0$；
- 纵轴：补贴持续时间 $T_s$；
- 颜色：最终综合份额 $L_A$。

可以划分区域：

| 区域 | 含义 |
|---|---|
| 无效补贴区 | 补贴太弱或太短，平台无法逆袭 |
| 临界补贴区 | 接近突破临界规模 |
| 成功锁定区 | 补贴退出后仍能保持优势 |

### 结果解释重点

可以强调：

- 补贴的目标不是永久提高吸引力，而是帮助平台跨过临界规模；
- 如果补贴时间太短，补贴退出后平台份额会回落；
- 如果补贴足够强且足够久，平台可能在补贴退出后依靠网络效应自我维持。

## 实验 10：利润约束实验

### 研究问题

如果考虑补贴成本，最优策略是否仍然是高补贴？

### 实验设置

定义平台 A 的即时利润：

$$
\Pi_A(t)=R_Au_A(t)m_A(t)-b_A^U(t)u_A(t)-b_A^M(t)m_A(t)-c_A
$$

其中：

- $R_Au_A(t)m_A(t)$ 表示平台交易收益，用户和商户都多时交易更多；
- $b_A^U(t)u_A(t)$ 是用户补贴成本；
- $b_A^M(t)m_A(t)$ 是商户补贴成本；
- $c_A$ 是固定运营成本。

累计利润：

$$
\Pi_A^{total}=\int_0^T \Pi_A(t)dt
$$

### 实现方式

1. 对不同补贴强度 $b$ 运行仿真；
2. 根据仿真得到的 $u(t),m(t)$ 计算每个时间点利润；
3. 使用数值积分计算累计利润；
4. 同时记录最终份额 $L_A$。

Python 可用：

```python
profit_t = R * u * m - bU * u - bM * m - c
profit_total = np.trapz(profit_t, t)
```

### 展示结果

建议展示双轴图或两张图：

1. 横轴补贴强度，纵轴最终份额 $L_A$；
2. 横轴补贴强度，纵轴累计利润 $\Pi_A^{total}$。

### 结果解释重点

要突出：

- 市场份额可能随补贴单调增加；
- 但利润不一定单调增加；
- 最优补贴应是能够跨过临界规模，同时不过度消耗利润的补贴水平。

## 实验 11：机器学习预测市场锁定

### 研究问题

能否根据模型参数预测市场是否会锁定？哪些因素最影响锁定结果？

### 数据生成

用仿真生成数据集。

每个样本的输入特征为：

$$
x=(\alpha,\beta,s_U,s_M,\Delta q,b_U,b_M,u_0,m_0)
$$

输出标签为：

$$
y=
\begin{cases}
1, & \text{市场锁定} \\
0, & \text{未锁定或共存}
\end{cases}
$$

也可以做三分类：

| 标签 | 含义 |
|---|---|
| 0 | 双平台共存 |
| 1 | 市场倾斜 |
| 2 | 赢家通吃 |

### 实验设置

随机采样参数范围：

| 参数 | 范围 |
|---|---|
| $\alpha$ | $[0,5]$ |
| $\beta$ | $[0,5]$ |
| $s_U,s_M$ | $[0,2]$ |
| $\Delta q$ | $[0,5]$ |
| $b_U,b_M$ | $[0,5]$ |
| $u_0,m_0$ | $[0.1,0.9]$ |

生成 2000 到 10000 组样本即可。

### 实现方式

1. 随机采样参数；
2. 对每组参数运行 ODE 仿真；
3. 根据最终 $C$ 或 $L_A$ 生成标签；
4. 训练分类模型。

可选模型：

- Logistic Regression；
- Random Forest；
- XGBoost；
- MLP。

建议优先使用 Random Forest，因为它实现简单且可以输出特征重要性。

### 展示结果

需要展示：

1. 分类准确率、F1-score；
2. 混淆矩阵；
3. 特征重要性柱状图；
4. 可选：二维参数平面上的决策边界。

### 结果解释重点

这个实验的重点不是追求最高准确率，而是解释：

- 哪些参数最影响市场锁定；
- 网络效应、初始优势、切换成本和补贴哪个更重要；
- 机器学习结果是否与前面的机制分析一致。

## 实验 12：消融与反事实实验

### 研究问题

不同机制对市场锁定的边际贡献是多少？如果去掉某个机制，市场结果会如何变化？

### 实验设置

设置完整模型为：

| 机制 | 是否保留 |
|---|---|
| 网络效应 | 是 |
| 切换成本 | 是 |
| 服务质量差异 | 是 |
| 补贴 | 是 |

然后分别构造消融模型：

| 模型版本 | 设置方式 | 目的 |
|---|---|---|
| Full Model | 保留全部机制 | 完整模型 |
| w/o Network | $\alpha=\beta=0$ | 去掉双边网络效应 |
| w/o Switching | $s_U=s_M=0$ | 去掉切换成本 |

### 实现方式

1. 选定一组代表性参数；
2. 分别运行完整模型和消融模型；
3. 记录最终市场集中度 $C$ 和平台 A 份额 $L_A$；
4. 比较不同版本的结果。

| 模型版本 | $u_A(\infty)$ | $m_A(\infty)$ | $C$ | $L_A$ |
|---|---:|---:|---:|---:|
| Full Model |  |  |  |  |
| w/o Network |  |  |  |  |
| w/o Switching |  |  |  |  |
| w/o Subsidy |  |  |  |  |
| w/o Quality |  |  |  |  |

也可以画柱状图展示 $C$ 的变化。

### 结果解释重点

可以写：

- 若去掉网络效应后锁定程度大幅下降，说明双边网络效应是市场锁定的主要原因；
- 若去掉切换成本后弱势平台更容易逆袭，说明锁定效应加剧了路径依赖；
- 若去掉补贴后平台 A 无法突破临界规模，说明补贴在早期竞争中具有干预作用。

## 实验 13：强化学习补贴策略实验

### 研究问题

平台能否自动学习补贴策略，使其在有限成本下打破市场锁定？

### 建模方式

将平台 A 视为智能体。

状态：

$$
s_t=(u_t,m_t,\alpha,\beta,s_U,s_M)
$$

动作：

$$
a_t=(b_U(t),b_M(t))
$$

为了简化，可以将动作离散化：

| 动作编号 | 用户补贴 | 商户补贴 |
|---|---:|---:|
| 0 | 0 | 0 |
| 1 | 低 | 0 |
| 2 | 0 | 低 |
| 3 | 中 | 中 |
| 4 | 高 | 高 |

奖励函数：

$$
R_t=\Delta L_A(t)-\eta(b_U(t)+b_M(t))
$$

其中：

$$
L_A(t)=\frac{u_A(t)+m_A(t)}{2}
$$

### 实现方式

简化版可以不用复杂 DQN，先做 tabular Q-learning：

1. 将 $u,m$ 离散成 10 个区间；
2. 状态为 $(u\_bin,m\_bin)$；
3. 每一步选择一个补贴动作；
4. 根据动态方程推进一个时间步；
5. 根据市场份额提升和补贴成本计算奖励；
6. 训练若干 episode。

如果时间不足，可以用网格搜索近似最优策略，作为“简化强化学习策略”。

### 对照策略

必须设置 baseline：

| 策略 | 含义 |
|---|---|
| 无补贴 | $b_U=b_M=0$ |
| 固定补贴 | 每期补贴相同 |
| 阶段性补贴 | 前 $T_s$ 期补贴，之后退出 |
| 学习型补贴 | 强化学习选择补贴动作 |

### 展示结果

展示：

1. 不同策略下的 $L_A(t)$ 曲线；
2. 不同策略下的累计补贴成本；
3. 不同策略下的累计收益或利润；
4. 强化学习策略在不同阶段选择的动作分布。

### 结果解释重点

这部分的核心结论可以是：

- 学习型策略不一定补贴最多，但可能在关键早期集中补贴；
- 当平台接近临界规模后，学习策略会减少补贴；
- 这说明补贴应作为跨越临界锁定门槛的阶段性工具，而不是长期固定支出。

## 实验 14：Agent-Based Model 多智能体仿真实验

### 研究问题

当用户和商户存在个体差异、随机偏好时，ODE 模型中的市场锁定结论是否仍然成立？

### 实验设置

构造：

- $N_U$ 个用户智能体；
- $N_M$ 个商户智能体；
- 两个平台 A/B。

每个用户 $i$ 选择平台 A 的效用：

$$
V_i^U(A)=q_A^U+\alpha m_A+b_A^U-p_A^U+\epsilon_i^U
$$

每个商户 $j$ 选择平台 A 的效用：

$$
V_j^M(A)=q_A^M+\beta u_A+b_A^M-r_A+\epsilon_j^M
$$

其中：

$$
\epsilon_i^U,\epsilon_j^M\sim N(0,\sigma^2)
$$

表示个体偏好噪声。

### 实现方式

每个离散时间步：

1. 根据当前平台份额计算每个用户和商户的效用；
2. 每个个体选择效用更高的平台，或按 Logit 概率选择；
3. 更新平台 A 的用户份额和商户份额；
4. 重复迭代。

伪代码：

```python
for t in range(T):
    for each user i:
        compute V_A, V_B
        choose platform by logit probability

    update u_A

    for each merchant j:
        compute V_A, V_B
        choose platform by logit probability

    update m_A
```

### 展示结果

建议展示：

1. ODE 模型与 Agent-Based Model 的 $u(t),m(t)$ 对比曲线；
2. 不同噪声水平 $\sigma$ 下的最终锁定程度；
3. 多次随机仿真的均值和方差。

### 结果解释重点

可以说明：

- 在个体偏好异质性存在时，市场锁定仍可能出现；
- 偏好噪声较大时，锁定过程可能变慢或不稳定；
- ODE 模型描述平均行为，Agent-Based Model 验证了结论的稳健性。
