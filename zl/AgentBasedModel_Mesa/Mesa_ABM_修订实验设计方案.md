# Mesa ABM 修订实验设计方案

## 1. 修订原则

当前 ABM 实验已经覆盖个体异质性、精准补贴、商户多归属、社交影响和冷启动策略，但实验设置仍偏向“展示现象”，缺少对临界区、对照公平性和机制边界的识别。因此，后续 ABM 实验不应继续追求实验数量，而应聚焦少数高含金量问题。

修订后的 ABM 实验应遵循四个原则：

1. **放在临界区**：避免参数过强导致所有结果都锁定，实验应设置在 A 可能赢、B 可能赢、也可能共存的边界区域。
2. **保证对照公平**：尤其是补贴实验，不同策略必须在相同预算下比较。
3. **寻找机制边界**：不只报告最终份额，而要找出锁定、共存、失败之间的临界条件。
4. **输出重点图表**：优先使用概率曲线、箱线图、热力图和相图，而不是单条最终份额曲线。

因此，ABM 部分建议从原来的五个实验收缩为三个主实验和一个可选扩展实验。

## 2. 实验组合调整

| 实验 | 是否作为主实验 | 调整意见 |
|---|---|---|
| 个体异质性 | 主实验 | 放到临界区，观察异质性是否改变锁定概率和共存概率 |
| 精准补贴 | 主实验 | 固定总预算，比较统一、随机、精准和双边精准补贴 |
| 冷启动 | 主实验 | 改成用户种子比例 × 商户种子比例二维热力图 |
| 商户多归属 | 可选扩展 | 重构有效商户供给机制，扫描多归属成本和有效供给贡献 |
| 社交影响 | 暂不作为主实验 | 需要社区内锁定和关键节点扩散指标，否则结果较难解释 |

推荐论文主线：

```text
临界区异质性实验
→ 等预算精准补贴实验
→ 冷启动二维资源配比实验
→ 商户多归属成本实验（可选）
```

这条主线可以形成清晰逻辑：

1. 个体差异会不会改变市场结构；
2. 平台能否通过更聪明的补贴跨过临界点；
3. 新平台冷启动需要怎样的双边资源组合；
4. 多归属机制是否能从制度层面削弱赢家通吃。

## 3. 主实验 A：临界区异质性实验

### 3.1 研究问题

个体偏好差异是否会让市场从赢家通吃转向双平台共存？

早期实验把平台 A 初始优势设得较强，导致 A 锁定概率始终为 1，最终只能得到“异质性降低 A 份额但不改变结局”的弱结论。修订后应把系统放到临界区，让市场存在多种可能结果。

### 3.2 实验设置

基础参数建议：

```text
u_A(0) = 0.52
m_A(0) = 0.52
alpha = beta = 2.2 或 2.5
sigma_user = sigma_merchant = 0.2
重复次数 = 50 或 100
仿真步数 = 80
```

扫描异质性强度：

\[
\sigma_{\theta}\in\{0,0.25,0.5,0.75,1.0,1.25,1.5,2.0\}
\]

其中 \(\sigma_{\theta}\) 控制用户和商户在以下属性上的差异：

- 价格敏感度；
- 补贴敏感度；
- 服务质量敏感度；
- 平台惯性；
- 商户流量敏感度；
- 商户抽成敏感度。

### 3.3 对照组

| 组别 | 设置 | 目的 |
|---|---|---|
| 同质 agent | 所有主体参数相同 | 代表 ODE 风格的平均主体 |
| 弱异质 agent | \(\sigma_\theta=0.25,0.5\) | 观察轻微差异是否影响锁定 |
| 强异质 agent | \(\sigma_\theta\ge 1.0\) | 观察强差异是否诱发共存 |

### 3.4 观察指标

核心指标：

\[
P(A\text{锁定}),\quad P(B\text{锁定}),\quad P(\text{共存})
\]

辅助指标：

- 最终 \(L_A\) 的均值；
- 最终 \(L_A\) 的方差；
- 市场集中度 \(C=L_A^2+(1-L_A)^2\)；
- 不同用户类型在 A/B 平台的分布；
- 不同商户类型在 A/B 平台的分布。

### 3.5 推荐图表

| 图表 | 说明 |
|---|---|
| 异质性强度 vs A/B/共存概率堆叠面积图 | 展示市场结构如何随异质性变化 |
| 异质性强度 vs 最终 \(L_A\) 箱线图 | 展示结果方差和路径分化 |
| 异质性强度 vs 市场集中度曲线 | 展示赢家通吃程度变化 |
| 类型分布堆叠图 | 展示不同类型主体如何分流到不同平台 |

### 3.6 预期结论

如果异质性较低，市场更容易被初始优势和网络效应推向单边锁定。

如果异质性较高，不同主体对价格、质量、补贴和惯性的反应不同，会形成差异化需求，削弱赢家通吃概率，并提高共存概率或结果方差。

该实验的论文价值在于回答：

> 个体差异不是简单降低领先平台份额，而是可能改变市场结构的稳定性。

## 4. 主实验 B：等预算精准补贴实验

### 4.1 研究问题

在相同补贴预算下，精准补贴是否比统一补贴更容易帮助弱势平台跨过临界规模？

早期实验中不同补贴策略的实际支出不同，因此“精准补贴优于统一补贴”的结论容易被质疑。修订后必须固定总预算。

### 4.2 实验设置

平台 A 为挑战者平台：

```text
u_A(0) = 0.30
m_A(0) = 0.30
alpha = beta = 2.5
heterogeneity_sigma = 0.8
重复次数 = 50 或 100
仿真步数 = 80
```

扫描总预算：

\[
B\in\{1000,3000,5000,8000,12000,16000\}
\]

每一种策略在每个预算水平下使用相同总预算 \(B\)。

### 4.3 策略对照

| 策略 | 预算是否相同 | 补贴对象 | 设计目的 |
|---|---|---|---|
| 无补贴 | 是 | 无 | 基准组 |
| 统一补贴 | 是 | 全体用户和商户平均分配 | 大水漫灌策略 |
| 随机补贴 | 是 | 随机抽取部分用户和商户 | 控制“少量对象”效应 |
| 摇摆用户补贴 | 是 | 当前在 B、低惯性、高补贴敏感用户 | 检验用户侧精准投放 |
| 关键商户补贴 | 是 | 高流量敏感、关键商户 | 检验供给侧精准投放 |
| 双边精准补贴 | 是 | 摇摆用户 + 关键商户 | 检验双边协同投放 |

### 4.4 补贴分配规则

固定预算约束：

\[
\sum_i b_i^U+\sum_j b_j^M = B
\]

统一补贴：

\[
b_i=\frac{B}{N_U+N_M}
\]

精准补贴：

\[
b_i=\frac{B}{|\mathcal{T}|},\quad i\in \mathcal{T}
\]

其中 \(\mathcal{T}\) 是被选中的目标 agent 集合。

### 4.5 观察指标

- 平台 A 成功概率：

\[
P(L_A(T)>0.6)
\]

- 平台 A 锁定概率；
- 平均跨越时间：

\[
\tau=\min\{t:L_A(t)>0.6\}
\]

- ROI：

\[
ROI=\frac{L_A(T)-L_A(0)}{B}
\]

- 临界预算：

\[
B^*=\min B,\quad P(L_A(T)>0.6)\ge 0.5
\]

### 4.6 推荐图表

| 图表 | 说明 |
|---|---|
| 预算 \(B\) vs A 成功概率曲线 | 识别不同策略的临界预算 |
| 预算 \(B\) vs ROI 曲线 | 比较单位预算效率 |
| 预算 \(B\) vs 平均跨越时间 | 比较扩散速度 |
| 不同策略下 \(L_A(t)\) 均值路径 | 展示策略动态过程 |

### 4.7 预期结论

精准补贴的价值不应体现为“花钱更多”，而应体现为在相同预算下更早跨过网络效应阈值。

理想结论形式：

> 双边精准补贴的临界预算低于统一补贴和随机补贴，说明平台竞争中的补贴效率取决于补贴对象识别，而不仅取决于补贴总量。

## 5. 主实验 C：冷启动二维资源配比热力图

### 5.1 研究问题

冷启动平台需要怎样的用户种子和商户种子组合，才能触发双边正反馈？

早期冷启动实验只比较少数固定策略，结果大多失败，无法看出边界在哪里。修订后应扫描用户侧和商户侧资源空间，寻找最小启动条件。

### 5.2 实验设置

平台 A 为新进入者或弱势平台。

扫描初始用户份额：

\[
u_A(0)\in\{0.05,0.10,0.15,0.20,0.25,0.30,0.35,0.40,0.45,0.50\}
\]

扫描初始商户份额：

\[
m_A(0)\in\{0.05,0.10,0.15,0.20,0.25,0.30,0.35,0.40,0.45,0.50\}
\]

也可以换成种子数量：

\[
S_U\times S_M
\]

基础参数：

```text
alpha = beta = 2.5 或 3.0
heterogeneity_sigma = 0.5
重复次数 = 30 或 50
仿真步数 = 80
```

### 5.3 观察指标

每个 \((u_A(0),m_A(0))\) 组合记录：

- 最终 \(L_A\) 均值；
- 平台 A 成功概率；
- 平台 A 锁定概率；
- 平均跨越 \(L_A>0.5\) 或 \(L_A>0.6\) 的时间；
- 失败概率。

### 5.4 推荐图表

| 图表 | 说明 |
|---|---|
| 用户种子比例 × 商户种子比例热力图，颜色为最终 \(L_A\) | 展示最终市场格局 |
| 用户种子比例 × 商户种子比例热力图，颜色为 A 成功概率 | 展示启动成功区域 |
| 等高线图 | 标出临界启动边界 |
| 固定商户种子比例下的用户种子切片图 | 展示单边资源边际效应 |

### 5.5 预期结论

双边平台冷启动通常不是单边资源越多越好，而是存在用户侧和商户侧的互补关系。

理想结论形式：

> 只有当用户种子和商户种子同时超过一定阈值时，平台 A 才能进入正反馈启动区；单独堆用户或单独堆商户都可能无法突破既有平台优势。

## 6. 扩展实验 D：商户多归属成本阈值实验

### 6.1 研究问题

降低商户多归属成本是否可以削弱平台锁定并提高市场共存概率？

早期多归属实验结果不够单调，主要原因是有效供给机制没有定义得足够清楚。修订后应明确多归属商户对两个平台的有效供给贡献。

### 6.2 模型机制

商户状态：

\[
A\text{-only},\quad B\text{-only},\quad Multi\text{-home}
\]

用户看到的平台有效供给定义为：

\[
M_A^{eff}=M_{A-only}+\rho M_{multi}
\]

\[
M_B^{eff}=M_{B-only}+\rho M_{multi}
\]

其中 \(\rho\in[0,1]\) 表示多归属商户对每个平台的有效供给贡献。

### 6.3 实验设置

扫描多归属成本：

\[
k_M\in\{0,0.2,0.5,0.8,1.2,1.6,2.0\}
\]

扫描有效供给贡献：

\[
\rho\in\{0.3,0.5,0.7,0.9,1.0\}
\]

基础参数建议设在临界区：

```text
u_A(0) = 0.52
m_A(0) = 0.52
alpha = beta = 2.2 或 2.5
重复次数 = 50
```

### 6.4 观察指标

- 多归属商户比例；
- 平台 A 锁定概率；
- 平台 B 锁定概率；
- 共存概率；
- 市场集中度；
- 用户可接触商户数量。

### 6.5 推荐图表

| 图表 | 说明 |
|---|---|
| 多归属成本 vs 多归属商户比例 | 检验成本机制是否合理 |
| 多归属成本 vs 共存概率 | 观察多归属是否削弱锁定 |
| \(k_M-\rho\) 热力图 | 展示制度参数如何影响市场结构 |
| 有效商户供给差异曲线 | 解释用户侧选择变化 |

### 6.6 预期结论

当多归属成本较低且多归属商户对两个平台都有较高有效供给贡献时，平台之间的商户供给差异被缩小，用户侧网络效应差距下降，市场更容易共存。

当多归属成本较高或 \(\rho\) 较低时，多归属机制弱化，市场更容易重新走向赢家通吃。

## 7. 可选实验 E：社交网络关键节点补贴

### 7.1 为什么不作为主实验

社交影响实验很有潜力，但如果只画总体 \(L_A\)，容易出现解释困难。例如 \(L_A\approx0.5\) 可能代表真正共存，也可能代表一半社区锁定 A、一半社区锁定 B。

因此，社交实验应从“社交影响强度扫描”改为“关键节点策略实验”。

### 7.2 研究问题

在用户存在社交影响时，补贴高影响力用户是否比随机补贴更有效？

### 7.3 策略对照

| 策略 | 补贴对象 |
|---|---|
| 随机用户补贴 | 随机选择用户 |
| 高度中心性用户补贴 | 选择 degree 或 PageRank 高的用户 |
| 社群桥接节点补贴 | 选择跨社区边较多的用户 |
| 价格敏感用户补贴 | 选择价格或补贴敏感用户 |

### 7.4 必须新增指标

- 社区内锁定率；
- 社区间分化程度；
- 高中心性用户带来的扩散速度；
- 不同社区最终平台选择分布。

### 7.5 推荐图表

| 图表 | 说明 |
|---|---|
| 策略 vs A 成功概率 | 比较关键节点补贴效果 |
| 策略 vs 扩散速度 | 比较传播效率 |
| 社区最终选择热力图 | 展示局部锁定 |
| 网络可视化快照 | 展示补贴节点和扩散路径 |

## 9. 代码修改建议

为了支持修订后的实验设计，建议对当前 `abm_experiment_model.py` 和 `run_abm_experiments.py` 做如下调整：

| 模块 | 修改内容 |
|---|---|
| 异质性实验 | 将初始份额改为 \(u_A(0)=m_A(0)=0.52\)，重复次数提高到至少 30 |
| 补贴实验 | 增加预算扫描 \(B\)，确保每种策略实际支出相同 |
| 补贴策略 | 新增随机补贴策略，作为精准补贴的公平对照 |
| 冷启动实验 | 改为二维网格扫描，输出热力图 |
| 多归属实验 | 明确 \(M_A^{eff},M_B^{eff}\) 的计算，并扫描 \(k_M,\rho\) |
| 社交实验 | 若保留，新增社区内锁定率和桥接节点补贴策略 |

## 10. 结论定位

修订后的 ABM 实验目标不是“证明模型会锁定”，而是识别：

```text
什么时候会锁定？
什么时候会共存？
什么策略能改变临界点？
冷启动至少需要多少双边资源？
多归属机制能否降低赢家通吃概率？
```

这会让 ABM 部分从“仿真展示”升级为“机制识别 + 策略评估”，图表和结论都会更适合放进最终论文。
