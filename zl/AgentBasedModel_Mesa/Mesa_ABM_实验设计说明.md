# Mesa ABM 实验设计说明

## 1. 设计思路

前期 ABM 实验主要用于对比 ODE 平均场模型，验证市场锁定、补贴门槛和随机波动等结论。这类实验可以说明 ABM 与 ODE 的一致性，但没有充分发挥 ABM 的优势。

ABM 更适合研究以下问题：

- 个体差异如何改变平台竞争结果；
- 平台是否应该对不同类型用户和商户采取差异化策略；
- 商户多归属是否会削弱平台锁定；
- 社交影响和局部网络如何放大路径依赖；
- 冷启动阶段的种子用户或种子商户如何影响长期格局。

因此，后续 ABM 实验应从“验证 ODE”转向“解释机制”和“比较策略”。本文档按照优先级设计五类更有价值的 ABM 实验。

## 2. 实验优先级总览

| 优先级 | ABM 实验问题 | 推荐理由 |
|---:|---|---|
| 1 | 个体异质性是否削弱赢家通吃 | 最适合 ABM，容易实现，结论有价值 |
| 2 | 精准补贴是否优于统一补贴 | 有 AI/算法策略特色，可解释为用户画像或策略投放 |
| 3 | 商户多归属是否削弱锁定 | 现实意义强，和平台竞争高度相关 |
| 4 | 社交影响是否加剧锁定 | 问题新颖，但实现稍复杂 |
| 5 | 种子用户/种子商户冷启动策略 | 适合作为扩展亮点 |

## 3. 实验一：个体异质性是否削弱赢家通吃

### 3.1 研究问题

当用户和商户存在不同偏好、价格敏感度、质量敏感度或平台忠诚度时，市场是否仍然容易形成赢家通吃？

### 3.2 实验动机

ODE 模型通常使用平均用户和平均商户，难以表达“有人偏好低价、有人偏好服务质量、有人更容易被补贴吸引、有人不愿切换平台”的差异。ABM 可以把这些差异直接放到个体 agent 中，因此这是最适合 ABM 的核心实验。

### 3.3 模型扩展

在现有 `UserAgent` 和 `MerchantAgent` 基础上，为每个 agent 增加个体属性：

```text
用户属性：
- price_sensitivity：价格敏感度
- quality_sensitivity：服务质量敏感度
- subsidy_sensitivity：补贴敏感度
- inertia：平台惯性或切换成本

商户属性：
- commission_sensitivity：抽成敏感度
- traffic_sensitivity：流量敏感度
- subsidy_sensitivity：补贴敏感度
- inertia：平台惯性或切换成本
```

异质性可以通过分布生成：

\[
\theta_i \sim N(\bar{\theta}, \sigma_\theta^2)
\]

或使用分群方式生成：

| 类型 | 特征 |
|---|---|
| 价格敏感型用户 | 对价格和补贴反应强 |
| 质量敏感型用户 | 更看重服务质量 |
| 惯性型用户 | 切换成本高，不容易换平台 |
| 流量敏感型商户 | 更看重用户规模 |
| 成本敏感型商户 | 更看重抽成和补贴 |

### 3.4 实验设计

设置异质性强度参数：

\[
\sigma_\theta \in \{0,0.25,0.5,0.75,1.0,1.5\}
\]

对每个异质性水平重复运行 30 到 100 次。

基础场景：

```text
平台 A 初始份额略高：u_A(0)=m_A(0)=0.55
平台 B 初始份额略低：u_B(0)=m_B(0)=0.45
两平台服务质量和价格基本相同
网络效应保持中高水平
```

对照组：

```text
同质 agent：所有用户和商户参数相同
异质 agent：用户和商户参数从分布中抽样
```

### 3.5 观察指标

- 平台 A 最终综合份额 \(L_A(\infty)\)；
- 平台 A 锁定概率；
- 平台 B 锁定概率；
- 双平台共存概率；
- 市场集中度：

\[
C=L_A^2+(1-L_A)^2
\]

- 不同用户类型在两平台上的分布；
- 不同商户类型在两平台上的分布。

### 3.6 预期结论

如果个体异质性较弱，网络效应更容易放大初始优势，市场更容易赢家通吃。

如果个体异质性较强，不同类型用户和商户会对平台形成差异化需求，部分用户和商户即使面对较小平台也可能留存，从而削弱赢家通吃并提高共存概率。

### 3.7 推荐图表

| 图表 | 含义 |
|---|---|
| 异质性强度 vs 平台 A 锁定概率 | 展示异质性是否削弱锁定 |
| 异质性强度 vs 市场集中度 | 展示市场是否更分散 |
| 用户类型在平台 A/B 的分布堆叠图 | 展示平台差异化吸引的人群 |
| 多次仿真的 \(L_A(t)\) 均值和置信区间 | 展示演化路径 |

## 4. 实验二：精准补贴是否优于统一补贴

### 4.1 研究问题

在补贴预算相同的情况下，平台应该对所有用户和商户统一补贴，还是根据 agent 类型进行精准补贴？

### 4.2 实验动机

现实平台不会只做简单统一补贴，而会根据用户画像、消费频率、价格敏感度、留存概率和商户价值进行差异化投放。ABM 可以自然模拟“给谁补、补多少、补多久”的策略问题，因此该实验具有较强 AI 和策略优化特色。

### 4.3 补贴策略设计

设平台 A 为挑战者平台，初始落后：

\[
u_A(0)=m_A(0)=0.30
\]

总补贴预算固定为 \(B\)。比较以下策略：

| 策略 | 说明 |
|---|---|
| 无补贴 | \(B=0\)，作为基准组 |
| 统一补贴 | 所有用户和商户获得相同补贴 |
| 用户侧精准补贴 | 只补贴价格敏感型或易转化用户 |
| 商户侧精准补贴 | 只补贴流量敏感型或头部商户 |
| 双边精准补贴 | 同时补贴关键用户和关键商户 |
| 动态精准补贴 | 根据每期份额和 agent 状态调整补贴 |

### 4.4 精准补贴规则示例

用户侧精准补贴：

```text
若用户 price_sensitivity 高，且当前在平台 B，则给予较高补贴；
若用户 inertia 高，则减少补贴，因为转化难度高；
若用户已经在平台 A，则只给予低额留存补贴。
```

商户侧精准补贴：

```text
优先补贴 traffic_sensitivity 高的商户；
优先补贴能吸引大量用户的头部商户；
对已经多次切换失败的商户减少补贴。
```

动态精准补贴：

```text
当 L_A < 0.4 时，优先补贴商户侧，建立供给；
当 0.4 <= L_A < 0.6 时，同时补贴用户和商户，跨越临界规模；
当 L_A >= 0.6 时，逐渐降低补贴，观察是否能自维持。
```

### 4.5 观察指标

- 平台 A 锁定概率；
- 达到 \(L_A>0.6\) 的概率；
- 达到 \(L_A>0.6\) 所需时间；
- 单位补贴带来的份额提升：

\[
ROI=\frac{L_A(T)-L_A(0)}{B}
\]

- 补贴浪费率：补贴给原本也会选择平台 A 的 agent 的预算比例；
- 不同策略下的预算使用路径。

### 4.6 预期结论

在总预算相同的情况下，精准补贴通常应优于统一补贴。尤其是在用户和商户存在明显异质性时，将补贴集中给高敏感度、低惯性、关键位置的 agent，更容易帮助挑战者平台跨过临界规模。

### 4.7 推荐图表

| 图表 | 含义 |
|---|---|
| 补贴策略 vs A 锁定概率 | 比较策略效果 |
| 补贴策略 vs ROI | 比较单位预算效率 |
| \(L_A(t)\) 路径图 | 展示不同策略的演化过程 |
| 预算使用堆叠图 | 展示补贴投向用户侧或商户侧 |

## 5. 实验三：商户多归属是否削弱锁定

### 5.1 研究问题

当商户可以同时入驻平台 A 和平台 B 时，市场是否更不容易形成赢家通吃？

### 5.2 实验动机

现实中的商户往往可以多平台经营，例如同时入驻多个外卖、电商或打车平台。商户多归属会削弱单个平台的独占优势，也会改变用户侧网络效应。因此，该实验与平台竞争高度相关。

### 5.3 模型扩展

当前模型中商户只能选择 A 或 B。需要扩展为三种状态：

```text
merchant.platform_status ∈ {"A_only", "B_only", "multi_home"}
```

商户选择多归属时，同时获得两个平台的用户流量，但需要承担额外运营成本：

\[
V_j^M(AB)=q_A^M+q_B^M+\beta \cdot f(u_A,u_B)-r_A-r_B-c_{multi}+\epsilon_j
\]

其中 \(c_{multi}\) 表示多归属成本。

用户侧网络效应需要调整：

```text
平台 A 对用户的吸引力 = A 独占商户份额 + 多归属商户份额的一部分
平台 B 对用户的吸引力 = B 独占商户份额 + 多归属商户份额的一部分
```

### 5.4 实验设计

扫描多归属成本：

\[
c_{multi}\in\{0,0.1,0.3,0.5,0.8,1.2\}
\]

也可以扫描允许多归属的商户比例：

\[
\rho_{multi}\in\{0,0.25,0.5,0.75,1.0\}
\]

实验组：

| 组别 | 说明 |
|---|---|
| 不允许多归属 | 当前基础模型 |
| 低成本多归属 | 多平台经营成本低 |
| 高成本多归属 | 多平台经营成本高 |
| 部分商户可多归属 | 只有一部分商户具备多平台运营能力 |

### 5.5 观察指标

- 平台 A/B 用户份额；
- A 独占商户比例；
- B 独占商户比例；
- 多归属商户比例；
- 市场集中度；
- 平台 A 或 B 锁定概率；
- 用户福利或可选商户数量。

### 5.6 预期结论

当多归属成本较低时，商户更倾向同时加入两个平台，用户侧感知到的商户差异缩小，平台之间的网络效应差距被削弱，赢家通吃概率下降。

当多归属成本较高时，商户仍会选择单一平台，市场更容易出现锁定。

### 5.7 推荐图表

| 图表 | 含义 |
|---|---|
| 多归属成本 vs 多归属商户比例 | 展示商户行为变化 |
| 多归属成本 vs 平台锁定概率 | 展示是否削弱赢家通吃 |
| A/B/多归属商户比例堆叠面积图 | 展示商户结构演化 |
| 用户可接触商户数量变化图 | 体现用户侧福利 |

## 6. 实验四：社交影响是否加剧锁定

### 6.1 研究问题

如果用户受到朋友、邻居或社群中其他用户选择的影响，平台锁定是否会更快、更强、更不稳定？

### 6.2 实验动机

现实中用户选择平台往往不是独立决策，而会受到社交关系、口碑传播、熟人推荐和群体行为影响。ABM 可以通过网络结构刻画这种局部互动，这是 ODE 平均场模型较难表达的机制。

### 6.3 模型扩展

为用户建立社交网络：

```text
network_type ∈ {"random", "small_world", "scale_free", "community"}
```

用户效用中增加社交影响项：

\[
V_i^U(A)=\cdots+\eta \cdot share_A(N_i)
\]

其中 \(share_A(N_i)\) 表示用户 \(i\) 的邻居中选择平台 A 的比例，\(\eta\) 表示社交影响强度。

### 6.4 实验设计

扫描社交影响强度：

\[
\eta\in\{0,0.2,0.5,1.0,1.5,2.0\}
\]

比较不同网络结构：

| 网络结构 | 含义 |
|---|---|
| 随机网络 | 用户关系随机 |
| 小世界网络 | 存在局部聚集和少量远距离连接 |
| 无标度网络 | 少数意见领袖或高连接用户影响较大 |
| 社群网络 | 用户分成多个相对独立的社群 |

### 6.5 观察指标

- 平台 A 锁定概率；
- 锁定所需时间；
- 不同社群的最终平台选择；
- 早期小扰动是否被放大；
- 意见领袖初始选择对最终结果的影响。

### 6.6 预期结论

社交影响较弱时，平台竞争主要由双边网络效应决定。社交影响较强时，局部从众会加速锁定过程，并可能导致不同社群锁定到不同平台。

在无标度网络中，少数高连接用户的选择可能对最终格局产生显著影响，因此种子用户选择变得更加重要。

### 6.7 推荐图表

| 图表 | 含义 |
|---|---|
| 社交影响强度 vs 锁定时间 | 展示是否加速锁定 |
| 社交影响强度 vs 锁定概率 | 展示是否加强赢家通吃 |
| 不同网络结构下的 \(L_A(t)\) 曲线 | 比较网络结构影响 |
| 社群最终选择热力图 | 展示局部锁定 |

## 7. 实验五：种子用户/种子商户冷启动策略

### 7.1 研究问题

平台在冷启动阶段应该优先获取种子用户，还是优先获取种子商户？不同种子选择策略对长期锁定结果有什么影响？

### 7.2 实验动机

双边平台冷启动的核心问题是“先有用户还是先有商户”。ABM 可以模拟少量种子 agent 对后续扩散的影响，因此该实验适合作为扩展亮点。

### 7.3 冷启动策略设计

设平台 A 为新进入者，初始份额较低：

\[
u_A(0)=m_A(0)=0.05
\]

比较以下种子策略：

| 策略 | 说明 |
|---|---|
| 随机种子用户 | 随机选择一批用户加入平台 A |
| 高影响力种子用户 | 选择社交网络中连接度高的用户 |
| 关键种子商户 | 选择能吸引大量用户的商户 |
| 用户优先 | 先投入用户侧，再吸引商户 |
| 商户优先 | 先投入商户侧，再吸引用户 |
| 双边平衡 | 同时投入用户和商户 |

### 7.4 实验设计

扫描种子规模：

\[
S_U,S_M\in\{10,25,50,100,200\}
\]

在总冷启动资源固定的情况下比较：

```text
全部投向用户侧
全部投向商户侧
用户和商户均分
按 30/70、70/30 比例分配
```

### 7.5 观察指标

- 平台 A 是否成功跨过临界规模；
- 达到 \(L_A>0.3,0.5,0.6\) 的时间；
- 最终平台 A 份额；
- 单个种子 agent 带来的长期份额贡献；
- 冷启动失败概率。

### 7.6 预期结论

如果用户侧网络结构中存在高影响力节点，优先获取种子用户可能更有效。如果商户供给对用户吸引力更关键，则优先获取关键商户可能更有效。

在多数情况下，双边平衡策略可能比单边投入更稳健，因为用户和商户之间存在相互强化的双边网络效应。

### 7.7 推荐图表

| 图表 | 含义 |
|---|---|
| 种子规模 vs 冷启动成功概率 | 展示资源门槛 |
| 种子策略 vs 最终 \(L_A\) | 比较策略优劣 |
| 种子策略 vs 达到临界规模时间 | 比较扩散速度 |
| 用户侧/商户侧资源分配热力图 | 找到最佳资源分配比例 |

## 8. 推荐实施顺序

建议先完成前三个实验，再根据时间选择第四和第五个实验。

| 顺序 | 实验 | 原因 |
|---:|---|---|
| 1 | 个体异质性是否削弱赢家通吃 | 实现成本最低，最能体现 ABM 与 ODE 的差异 |
| 2 | 精准补贴是否优于统一补贴 | 结论最适合写成策略建议，也有 AI 特色 |
| 3 | 商户多归属是否削弱锁定 | 现实意义强，适合作为论文重点实验 |
| 4 | 社交影响是否加剧锁定 | 新颖，但需要引入网络结构 |
| 5 | 种子用户/种子商户冷启动策略 | 可作为扩展实验或附加亮点 |

## 9. 与现有代码的关系

现有模型已经支持：

- 用户和商户两类 agent；
- 平台 A/B 选择；
- 双边网络效应；
- 补贴、价格、抽成；
- 切换惯性；
- 个体偏好噪声；
- 多次随机仿真和可视化。

为了完成上述ABM 实验，建议进一步扩展：

| 扩展内容 | 影响文件 | 说明 |

## 10. 论文写作建议

ABM 部分可以从“补充验证”升级为“机制实验”。建议论文中这样组织：

1. 先说明 ODE 模型刻画平均市场份额，但难以表达个体异质性、精准策略、多归属和社交传播。
2. 引入 ABM，用用户和商户 agent 表示微观主体。
3. 重点展示三个核心实验：
   - 个体异质性削弱赢家通吃；
   - 精准补贴优于统一补贴；
   - 商户多归属削弱平台锁定。
4. 将社交影响和冷启动策略作为扩展实验或鲁棒性分析。

最终可以形成更有价值的结论：

> 平台竞争中的赢家通吃并非只由平均网络效应决定，还受到个体偏好结构、补贴投放策略、商户多归属能力和局部社交传播的共同影响。ABM 能够揭示这些微观机制如何累积为宏观市场格局。
