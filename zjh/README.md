# 双边平台竞争中的网络效应与市场锁定建模实验说明

## 1. 项目概述

本项目围绕“**双边平台竞争中的网络效应与市场锁定**”建立一套完整的数学建模与仿真实验框架。研究对象可以对应外卖平台、电商平台、网约车平台、在线内容平台等典型双边市场。在这类市场中，平台同时连接两类参与者，例如：

- 外卖平台：用户与商户；
- 电商平台：消费者与卖家；
- 网约车平台：乘客与司机；
- 内容平台：观众与创作者。

双边平台的核心特征是**交叉网络效应**：一侧规模越大，另一侧加入平台的意愿越强。例如用户越多，商户越愿意入驻；商户越多，用户也越愿意使用平台。这种正反馈可能使多个平台长期共存，也可能导致某一个平台通过早期优势逐渐形成“赢家通吃”的市场锁定。

本项目通过六个阶段的实验，从宏观动态模型到微观智能体模型，系统分析以下问题：

1. 双边网络效应如何导致市场锁定；
2. 初始用户规模和初始商户规模如何影响长期结果；
3. 服务质量优势能否帮助弱势平台打破锁定；
4. 用户补贴、商户补贴、双边补贴的效果有何不同；
5. 供给不足是否会削弱用户补贴效果；
6. 服务质量投资在长期竞争中是否有效；
7. 参数变化是否会改变最优策略；
8. 在引入异质性用户、异质性商户、多归属和随机偏好后，宏观模型结论是否仍然稳健。

---

## 2. 总体建模思路

本项目采用两层模型结构：

1. **宏观双边 Logit 动态模型**；
2. **Mesa Agent-Based Model 微观智能体模型**。

两类模型的定位不同。

宏观双边 Logit 动态模型主要用于刻画总体市场份额的演化。它直接研究平台 A 的用户份额 \(x(t)\) 和商户份额 \(y(t)\) 如何随时间变化，适合做参数扫描、热力图、临界阈值、策略比较和成本收益分析。

Mesa Agent-Based Model 则从微观个体角度出发，将用户和商户建模为具有不同偏好的智能体。它可以刻画宏观模型中难以直接表示的因素，例如价格敏感型用户、质量敏感型用户、惯性用户、大商户、中小商户、新商户、多归属商户、随机偏好扰动等。Mesa 模型主要用于验证宏观模型结论在异质性环境下是否仍然成立。

整体逻辑为：

```text
宏观模型：解释双边网络效应、市场锁定、补贴和投资机制
        ↓
参数敏感性：验证结论是否依赖某一组固定参数
        ↓
Mesa 模型：从微观异质性和多归属角度补充验证
```

## 4. 环境配置与运行方法

### 4.1 Python 版本

建议使用 Python 3.10 及以上版本。若使用新版 Mesa，建议 Python 3.11 或 Python 3.12。

查看当前 Python 版本：

```bash
python --version
```

项目主要依赖：

```txt
numpy
pandas
matplotlib
mesa
networkx
```

如果已有 `requirements.txt`，可运行：

```bash
python -m pip install -r requirements.txt
```

也可以单独安装：

```bash
python -m pip install numpy pandas matplotlib mesa networkx
```

如果使用 conda，可新建环境：

```bash
conda create -n platform_model python=3.11 -y
conda activate platform_model
python -m pip install numpy pandas matplotlib mesa networkx
```

在项目根目录下运行：

```bash
python main.py
```

程序会依次执行阶段 1 至阶段 6，并将图像和数据保存到：

```text
outputs_full_logit/
```

调试时不建议每次都运行全部实验，尤其 Mesa 阶段计算量较大。可以临时修改 `main.py`。

例如只运行阶段 6：

```python
from experiments.stage6_mesa_abm import run_stage6
from src.utils import ensure_dir


def main():
    output_root = "outputs_full_logit"
    ensure_dir(output_root)
    run_stage6(output_root)


if __name__ == "__main__":
    main()
```

## 5. 宏观双边 Logit 动态模型

### 5.1 状态变量

模型研究两个竞争平台 A 和 B。

平台 A 的用户份额为：

\[
x(t)\in[0,1]
\]

平台 A 的商户份额为：

\[
y(t)\in[0,1]
\]

因此平台 B 的用户份额和商户份额分别为：

\[
1-x(t),\qquad 1-y(t)
\]

其中：

| 符号 | 含义 |
|---|---|
| \(x(t)\) | 平台 A 在用户侧的市场份额 |
| \(y(t)\) | 平台 A 在商户侧的市场份额 |
| \(1-x(t)\) | 平台 B 在用户侧的市场份额 |
| \(1-y(t)\) | 平台 B 在商户侧的市场份额 |

---

### 5.2 核心参数

宏观模型的主要参数集中在 `src/config.py` 的 `Params` 类中。

| 参数 | 含义 |
|---|---|
| `alpha` / \(\alpha\) | 商户规模对用户选择平台的影响 |
| `beta` / \(\beta\) | 用户规模对商户选择平台的影响 |
| `lambda_u` | 用户侧 Logit 敏感度 |
| `lambda_m` | 商户侧 Logit 敏感度 |
| `adjust_u` / `r_U` | 用户份额向目标份额调整的速度 |
| `adjust_m` / `r_M` | 商户份额向目标份额调整的速度 |
| `q_u_A` | 平台 A 对用户侧的服务质量优势 |
| `q_m_A` | 平台 A 对商户侧的服务质量优势 |
| `subsidy_user_A` | 平台 A 对用户的补贴 |
| `subsidy_merchant_A` | 平台 A 对商户的补贴 |
| `shortage_enabled` | 是否启用供给不足惩罚 |
| `shortage_rho` / \(\rho\) | 供给不足惩罚强度 |
| `shortage_buffer` / \(\varepsilon\) | 供给不足缓冲项 |
| `supply_capacity` / \(\kappa\) | 单位商户供给能力 |
| `invest_eff_u` / \(\mu_U\) | 用户侧服务质量投资效率 |
| `invest_eff_m` / \(\mu_M\) | 商户侧服务质量投资效率 |
| `quality_decay` / \(\delta\) | 服务质量优势衰减率 |

其中最重要的是 \(\alpha\) 和 \(\beta\)。

\[
\alpha
\]

表示商户规模对用户的吸引作用。如果 \(\alpha\) 大，说明用户很看重平台上商户是否足够多。

\[
\beta
\]

表示用户规模对商户的吸引作用。如果 \(\beta\) 大，说明商户很看重平台上是否有足够多用户。

---

### 5.3 用户侧效用

用户选择平台 A 时，主要考虑：

1. 平台 A 的商户规模；
2. 平台 A 的用户侧服务质量；
3. 平台 A 的用户补贴；
4. 供给不足带来的体验损失。

可以抽象为：

\[
U_A^U
=
\alpha y
+
q_A^U
+
s_A^U
-
\rho \cdot \text{shortage}_A
\]

平台 B 的用户效用可以类似表示为：

\[
U_B^U
=
\alpha(1-y)
+
q_B^U
+
s_B^U
-
\rho \cdot \text{shortage}_B
\]

用户选择平台 A 的目标概率为：

\[
P_A^U
=
\frac{1}{1+\exp[-\lambda_U(U_A^U-U_B^U)]}
\]

---

### 5.4 商户侧效用

商户选择平台 A 时，主要考虑：

1. 平台 A 的用户规模；
2. 平台 A 的商户侧服务质量；
3. 平台 A 的商户补贴；
4. 佣金或经营成本。

可以抽象为：

\[
U_A^M
=
\beta x
+
q_A^M
+
s_A^M
\]

平台 B 的商户效用为：

\[
U_B^M
=
\beta(1-x)
+
q_B^M
+
s_B^M
\]

商户选择平台 A 的目标概率为：

\[
P_A^M
=
\frac{1}{1+\exp[-\lambda_M(U_A^M-U_B^M)]}
\]

---

### 5.5 市场份额动态方程

市场份额并不是瞬间跳到 Logit 选择概率，而是逐渐向目标份额调整：

\[
\frac{dx}{dt}
=
r_U(P_A^U-x)
\]

\[
\frac{dy}{dt}
=
r_M(P_A^M-y)
\]

其中 \(r_U\)、\(r_M\) 分别表示用户侧和商户侧的调整速度。

该动态方程体现了市场演化中的惯性：即使某一平台当前吸引力更高，市场份额也需要一段时间才能完成迁移。

---

### 5.6 供给不足惩罚

在阶段 3 之后，模型加入供给不足惩罚。平台 A 的供给不足定义为：

\[
\text{shortage}_A
=
\max(0,x-\kappa y-\varepsilon)
\]

其中：

| 参数 | 含义 |
|---|---|
| \(x\) | 用户需求规模 |
| \(y\) | 商户供给规模 |
| \(\kappa\) | 单位商户供给能力 |
| \(\varepsilon\) | 缓冲项 |

当用户份额增长过快，而商户份额没有同步增长时，\(\text{shortage}_A\) 会变大，从而降低用户效用。

这用于模拟现实中的情况，例如：

- 外卖平台用户增长过快，但商户和骑手不足；
- 网约车平台乘客增加，但司机不足；
- 电商平台流量增加，但优质卖家不足；
- 内容平台观众增加，但优质创作者不足。

---

### 5.7 服务质量投资

阶段 4 之后，服务质量不再只是固定参数，而是由平台投资动态积累：

\[
\frac{dq_U}{dt}=\mu_U I_U-\delta q_U
\]

\[
\frac{dq_M}{dt}=\mu_M I_M-\delta q_M
\]

其中：

| 参数 | 含义 |
|---|---|
| \(q_U\) | 用户侧服务质量优势 |
| \(q_M\) | 商户侧服务质量优势 |
| \(I_U\) | 用户侧服务质量投资 |
| \(I_M\) | 商户侧服务质量投资 |
| \(\mu_U,\mu_M\) | 投资效率 |
| \(\delta\) | 服务质量衰减率 |

如果投资效率高、衰减率低，服务质量优势容易积累并形成长期竞争壁垒；如果投资效率低、衰减率高，服务质量投资可能难以打破锁定，甚至因成本较高导致净收益下降。

## 6. 策略模块设计

策略主要写在 `src/policies.py` 中。每个策略根据时间 \(t\)、当前市场状态和参数，决定平台 A 当前的补贴和服务质量投资。

### 6.1 无策略

无策略表示平台 A 不采取补贴，也不进行额外服务质量投资。

```text
用户补贴 = 0
商户补贴 = 0
服务质量投资 = 0
```

它用于作为基准组。

### 6.2 用户补贴策略

用户补贴主要用于吸引用户侧参与者。常用形式为指数衰减：

\[
s_U(t)=s_{U0}e^{-kt}
\]

其中：

| 参数 | 含义 |
|---|---|
| \(s_{U0}\) | 初始用户补贴强度 |
| \(k\) | 补贴衰减率 |

用户补贴可以快速提高用户侧份额，但如果商户侧没有同步扩张，可能导致供给不足。

### 6.3 商户补贴策略

商户补贴主要用于吸引商户入驻：

\[
s_M(t)=s_{M0}e^{-kt}
\]

商户补贴先扩大供给侧，再通过商户规模提高用户效用。它在存在供给不足约束时通常比单纯用户补贴更稳健。

### 6.4 双边补贴策略

双边补贴同时对用户和商户补贴：

\[
s_U(t)=s_{U0}e^{-kt}
\]

\[
s_M(t)=s_{M0}e^{-kt}
\]

双边补贴更协调，但成本也更高。若单边补贴已经足以跨过临界规模，双边补贴可能出现成本冗余。

### 6.5 贪心服务质量投资策略

贪心策略在前期集中投入较强的补贴或服务质量投资，目标是快速跨过临界规模。

特点：

- 前期增长快；
- 累计净收益可能较高；
- 可能造成供给不足；
- 对投资效率较敏感。

### 6.6 长期服务质量投资策略

长期策略持续进行服务质量投资，目标是逐步积累质量优势。

特点：

- 增长较慢；
- 稳定性较好；
- 供给不足风险低；
- 成本较高。

### 6.7 动态策略

动态策略根据当前市场状态调整投入方向。例如：

- 用户份额较低时，加强用户补贴；
- 商户份额较低时，加强商户补贴；
- 出现供给不足时，优先补贴商户或提升商户侧服务质量；
- 市场份额稳定后，减少补贴，转向服务质量投资。

动态策略的优点是更灵活，缺点是成本可能较高，需要进一步调参优化。

## 7. 六个阶段的完整实验设计

## 7.1 阶段 1：基础模型实验

对应文件：

```text
experiments/stage1_basic.py
```

### 实验目的

阶段 1 用于验证基础双边网络效应是否会产生市场锁定，并分析初始规模差异和服务质量优势的影响。

### 实验 1：网络效应与初始规模耦合

固定平台 A 和平台 B 除初始份额外完全对称，扫描平台 A 的初始用户份额和初始商户份额：

\[
x_0\in[0.05,0.95]
\]

\[
y_0\in[0.05,0.95]
\]

考察不同网络效应强度下最终 \(x^*\) 的热力图。

典型参数：

| 参数 | 设置 |
|---|---:|
| \(x_0\) | 0.05 到 0.95 网格扫描 |
| \(y_0\) | 0.05 到 0.95 网格扫描 |
| \(\alpha=\beta\) | 0.2, 0.6, 1.0, 1.4 |
| 仿真时间 \(T\) | 60 左右 |
| 时间步长 \(dt\) | 0.04 或 0.05 |
| 补贴 | 无 |
| 服务质量差异 | 无 |

实验关注：

1. 网络效应弱时是否双平台共存；
2. 网络效应强时是否形成赢家通吃；
3. 初始份额是否决定最终市场归属。

### 实验 2：服务质量能否打破锁定

设置平台 A 初始处于劣势：

\[
x_0=0.35,\qquad y_0=0.35
\]

逐渐提高平台 A 的服务质量优势：

\[
q=\Delta q_U=\Delta q_M
\]

观察平台 A 是否能反超。

典型参数：

| 参数 | 设置 |
|---|---:|
| \(x_0\) | 0.35 |
| \(y_0\) | 0.35 |
| \(\alpha\) | 约 1.2 |
| \(\beta\) | 约 1.0 或 1.2 |
| \(q\) | 0 到 1.2 扫描 |
| 锁定判据 | \(x^*\geq 0.8\) |
| 反超判据 | \(x^*\geq 0.5\) |

阶段 1 的主要结论：

> 网络效应越强，市场越容易锁定；服务质量可以打破锁定，但需要超过临界阈值。

## 7.2 阶段 2：加入补贴策略

对应文件：

```text
experiments/stage2_subsidy.py
```

### 实验目的

阶段 2 分析补贴能否帮助弱势平台打破锁定，并比较用户补贴、商户补贴和双边补贴的成本收益差异。

### 初始设置

平台 A 初始处于劣势：

\[
x_0=0.35,\qquad y_0=0.35
\]

典型参数：

| 参数 | 设置 |
|---|---:|
| \(x_0\) | 0.35 |
| \(y_0\) | 0.35 |
| \(\alpha\) | 1.2 左右 |
| \(\beta\) | 1.0 左右 |
| 仿真时间 \(T\) | 80 |
| 时间步长 \(dt\) | 0.04 |
| 用户补贴初值 | 0.85 |
| 商户补贴初值 | 0.85 |
| 双边补贴初值 | 用户侧 0.55，商户侧 0.55 |
| 补贴衰减率 | 0.06 左右 |

### 比较策略

| 策略 | 说明 |
|---|---|
| 无补贴 | 基准组 |
| 用户补贴 | 只对用户侧补贴 |
| 商户补贴 | 只对商户侧补贴 |
| 双边补贴 | 同时补贴用户和商户 |

### 输出指标

| 指标 | 含义 |
|---|---|
| 最终用户份额 \(x^*\) | 平台 A 最终用户侧结果 |
| 最终商户份额 \(y^*\) | 平台 A 最终商户侧结果 |
| 累计收益 | 平台 A 从用户和商户获得的收益 |
| 累计补贴成本 | 平台补贴支出 |
| 累计净收益 | 收益减去成本 |

阶段 2 的主要结论：

> 补贴可以帮助平台跨过临界规模，但不是补贴越多越好。双边补贴更稳健，但成本较高；单边补贴若足以触发正反馈，可能具有更高净收益。

## 7.5 阶段 5：参数敏感性与拓展分析

阶段 5 用于验证前四阶段结论是否依赖特定参数，并进一步分析策略选择在不同市场环境中的变化。

### 拓展 1：非对称网络效应下的补贴方向选择

扫描：

\[
\alpha\in[0.3,1.8]
\]

\[
\beta\in[0.3,1.8]
\]

比较用户补贴、商户补贴和双边补贴的累计净收益，得到不同 \((\alpha,\beta)\) 下的最优补贴方向。

| 选择标准 | 累计净收益最大 |

核心结论：

> 当 \(\alpha>\beta\) 时，商户补贴更可能有效；当 \(\beta>\alpha\) 时，用户补贴更可能有效。

### 拓展 2：最低有效补贴组合

目标为：

\[
\min_{s_U,s_M}(s_U+s_M)
\]

约束为：

\[
x^*\geq 0.8,\qquad y^*\geq 0.8
\]

扫描初始用户补贴和商户补贴组合：

\[
s_U\in[0,1.2]
\]

\[
s_M\in[0,1.2]
\]

典型市场结构：

| 市场结构 | \(\alpha\) | \(\beta\) |
|---|---:|---:|
| 对称强网络效应 | 1.2 | 1.2 |
| 用户更依赖商户 | 1.6 | 0.5 |
| 商户更依赖用户 | 0.5 | 1.6 |
| 极强锁定市场 | 1.8 | 1.8 |

核心结论：

> 网络效应越强，最低有效补贴越高；补贴方向应匹配网络效应结构。

### 拓展 3：供给不足惩罚强度敏感性

扫描：

\[
\rho\in[0,10]
\]

比较用户补贴、商户补贴和双边协调补贴。

典型参数：

|---|---:|
| \(\rho\) | 0 到 10 |
| \(x_0\) | 0.35 |
| \(y_0\) | 0.35 |
| \(\alpha\) | 1.2 |
| \(\beta\) | 1.0 |

核心结论：

> 用户补贴对供给不足惩罚高度敏感。当 \(\rho\) 较大时，用户补贴可能突然失效；商户补贴和双边协调补贴更稳健。

### 拓展 4：服务质量投资效率与衰减率敏感性

扫描：

\[

\]

\[
\delta\in[0.01,0.10]

核心结论：

> 服务质量投资存在效率门槛。高投资效率、低衰减率有利于形成长期竞争优势；低效率、高衰减时投资可能失败甚至亏损。

## 9. 结果解释逻辑

本项目的分析逻辑如下：

1. 阶段 1 说明双边网络效应本身能够导致锁定；

3. 阶段 3 说明补贴必须考虑供给约束；
4. 阶段 4 说明服务质量投资是长期竞争方式；
5. 阶段 5 说明策略选择依赖参数环境；

整体结论是：

> 平台竞争结果不是由单一因素决定的，而是由网络效应强度、初始规模、补贴方向、供给能力、服务质量投资效率、用户和商户异质性共同决定。

## 12. 总结

本项目通过宏观 Logit 动态模型和 Mesa 微观智能体模型，系统分析了双边平台竞争中的网络效应、市场锁定、补贴策略、供给不足、服务质量投资、异质性和多归属行为。实验结果表明，平台竞争的关键不是简单地“增加补贴”，而是要根据市场中用户侧和商户侧的相互依赖关系，选择合适的补贴方向和长期投资策略。

对于弱势平台而言，单纯用户补贴并不总是稳健。更合理的策略是结合商户侧供给建设、服务质量提升和动态补贴调整，在短期扩张、长期质量、供需协调和成本控制之间取得平衡。
